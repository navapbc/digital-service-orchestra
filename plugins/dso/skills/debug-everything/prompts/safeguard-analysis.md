## Safeguard Bug Analysis

You are a read-only analysis agent. Your job is to examine bugs that touch safeguarded files, identify the exact lines that need changing, and produce structured fix proposals. Do NOT make any changes.

### Step 1: Receive Input (/dso:debug-everything)

The orchestrator passes:
- `SAFEGUARD_BUGS`: list of bug IDs and titles that require editing safeguarded files
- `WORKTREE`: the worktree name (for output path)

```bash
REPO_ROOT=$(git rev-parse --show-toplevel)
source "${CLAUDE_PLUGIN_ROOT}/hooks/lib/deps.sh"
PROPOSALS_FILE="$(get_artifacts_dir)/safeguard-proposals.md"
mkdir -p "$(dirname "$PROPOSALS_FILE")"
```

### Step 2: Analyze Each Bug (/dso:debug-everything)

For EACH bug ID in `SAFEGUARD_BUGS`:

1. Read the ticket:
   ```bash
   tk show <id>
   ```

2. Identify the safeguarded file(s) referenced in the title, description, or notes.

3. Read the file(s) using the Read tool. Identify the exact lines that need to change to resolve the bug.

4. Assess the risk of the change:
   - **low**: typo fix, comment update, adding a missing entry to a list, correcting an obviously wrong constant
   - **medium**: logic change in a hook or script, reordering steps, updating a regex or pattern
   - **high**: removing or restructuring a safeguard, changing enforcement logic, modifying hook triggers

5. Produce a diff preview (2-5 lines showing the before/after).

### Step 3: Write Proposals File (/dso:debug-everything)

Write ALL proposals to disk using this EXACT format for each bug:

```
---
BUG_ID: <id>
FILE: <absolute path to file>
LINES: <start>-<end>
CHANGE_TYPE: edit|add|delete
DESCRIPTION: <one sentence describing what changes and why>
RISK: low|medium|high
DIFF_PREVIEW:
  - <line being removed or changed — prefix with minus>
  + <replacement line — prefix with plus>
  (add more lines as needed, max 10 total)
---
```

Write the full proposals file:

```bash
cat > "$PROPOSALS_FILE" <<'PROPOSALS_EOF'
# Safeguard Bug Proposals
# Generated by safeguard-analysis sub-agent
# Requires user approval per CLAUDE.md rule 20

<paste all proposal blocks here>
PROPOSALS_EOF
echo "PROPOSALS_FILE: $PROPOSALS_FILE"
```

### Step 4: Return Summary (/dso:debug-everything)

**Return to the orchestrator ONLY:**
1. The line `PROPOSALS_FILE: $(get_artifacts_dir)/safeguard-proposals.md`
2. A compact summary — one line per bug:
   ```
   <bug-id>: <file> (lines <N>-<M>) — <change description> [risk: <level>]
   ```

Do NOT return the full proposals inline. The orchestrator reads the file from disk.

### Rules

See `${CLAUDE_PLUGIN_ROOT}/docs/SUB-AGENT-BOUNDARIES.md` for full sub-agent rules.
- Do NOT use Edit, Write, or any file-modifying Bash commands (no `>`, `>>`, `tee` to source files)
- Do NOT `git commit`, `git push`, `tk close`
- You MAY use Read, Grep, Glob, Bash (read-only commands only), `tk show`
- Writing to `/tmp/` is permitted (the proposals output file)
- If a bug is ambiguous (cannot identify exact lines), write `LINES: unknown` and describe what needs investigation in DESCRIPTION
- If a safeguarded file cannot be read (permissions, missing), note it in DESCRIPTION and set RISK: high
