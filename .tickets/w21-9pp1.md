# As a DSO practitioner, ESCALATED investigation uses four specialized agents with empirical veto and resolution

## Description

**What**: ESCALATED investigation tier — four opus agents with specialized roles, empirical veto power, resolution agent on conflict, and terminal escalation to user.
**Why**: When ADVANCED investigation fails, the four-agent ESCALATED mode attacks from maximally diverse angles. The empirical/logging agent provides ground truth that can veto theoretical consensus.
**Scope**:
- IN: Four ESCALATED prompt templates — (1) web researcher (error pattern analysis, similar issue correlation, dependency changelogs), (2) history analyst (timeline reconstruction, fault tree analysis, commit bisection), (3) code tracer (execution path tracing, dependency-ordered reading, intermediate variable tracking, five whys), (4) empirical/logging agent (authorized to add logging/debugging, validates or vetoes hypotheses from agents 1-3); veto logic, resolution agent dispatch on conflict, terminal escalation to user when no high-confidence root cause survives
- OUT: None — this is the final investigation tier

## Done Definitions

- When this story is complete, an ESCALATED investigation dispatches four specialized opus agents, each with a differentiated analytical lens
  ← Satisfies: "ESCALATED: four opus agents"
- When this story is complete, Agent 4 (empirical/logging) can veto hypotheses from agents 1-3, triggering a resolution agent to weigh all findings and conduct additional tests
  ← Satisfies: "if Agent 4 vetoes consensus, a resolution agent weighs all findings"
- When this story is complete, if ESCALATED investigation fails to produce a high-confidence root cause, all findings are surfaced to the user with no blind fix attempt
  ← Satisfies: "When ESCALATED investigation also fails to produce a high-confidence root cause, the skill surfaces all findings and escalates to the user"
- When this story is complete, escalation to the next investigation tier triggers when (a) fix verification fails or (b) investigation returns no/low-confidence root cause
  ← Satisfies: "Escalation to the next investigation tier triggers when..."

## Considerations

- [Reliability] Veto/resolution logic must handle edge cases: all agents disagree, Agent 4 times out, resolution agent also disagrees
- [Testing] Testable with mock agent responses for each of the four roles
- [Maintainability] Prompts extend shared base with role-specific techniques
- [Maintainability] Agent 4's logging/debugging additions must be reverted or stashed after evidence is collected — investigation artifacts must not persist in the codebase. Document findings in the investigation report, not in code

## Escalation Policy

**Escalation policy**: Proceed unless a significant assumption is required to continue — one that could send the implementation in the wrong direction. Escalate only when genuinely blocked without a reasonable inference. Document all assumptions made without escalating.
