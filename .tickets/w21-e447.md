# dso-setup.sh supplement_template_file uses cat for append — not in _make_tool_path fake PATH


## Notes

**2026-03-20T01:32:44Z**

Discovered during w21-8qo1 implementation. The supplement_template_file() function in dso-setup.sh uses 'cat' to append template content to existing files (the supplement path). The _make_tool_path helper in tests/scripts/test-dso-setup.sh does not include 'cat' in its needed_tools list. Currently this is not a problem because restricted-PATH tests do not pre-populate CLAUDE.md without DSO markers (so the append/cat path is never triggered in prereq tests). However, future tests that combine restricted PATH + existing file without markers would fail with command-not-found for cat. Consider replacing 'cat' with a bash built-in approach (read/while loop) to eliminate the external dependency.
