Extend plugins/dso/scripts/bridge-inbound.py to implement all remaining functions tested by w21-w5vc. After this task the script is complete and fully functional.

IMPLEMENTATION REQUIREMENTS:

1. map_status(jira_status: str, mapping: dict) -> str | None
   - Looks up jira_status in mapping dict; returns local status string or None if unmapped

2. map_type(jira_type: str, mapping: dict) -> str | None
   - Same pattern as map_status

3. write_bridge_alert(ticket_id: str, reason: str, tickets_root: Path, bridge_env_id: str) -> Path
   - Writes BRIDGE_ALERT event file: <epoch>-<uuid>-BRIDGE_ALERT.json
   - Fields: event_type=BRIDGE_ALERT, reason=reason, env_id=bridge_env_id, timestamp=epoch

4. fetch_jira_changes (EXTEND from T2): add full pagination
   - Loop: call acli_client.search_issues(jql, start_at, max_results=100) until response is empty
   - Accumulate all pages into flat list

5. verify_jira_timezone_utc(acli_client) -> bool
   - Calls acli_client.get_server_info() and checks timeZone == 'UTC'
   - Returns False (with warning log) if not UTC; True if UTC
   - Fast-abort caller (process_inbound) halts workflow startup if False

6. process_inbound(tickets_root, acli_client, checkpoint_path: Path, config: dict) -> None
   - Entry point: verify_jira_timezone_utc (halt if fails) -> read last_pull_ts from checkpoint -> fetch_jira_changes with config.get('overlap_buffer_minutes', 15) -> for each issue: normalize_timestamps -> check status/type mapping -> write_bridge_alert if unmapped -> write_create_events for new Jira-originated tickets -> update checkpoint ONLY on success (dual-timestamp checkpoint: do NOT advance on partial failure)
   - checkpoint_path: JSON file with {'last_pull_timestamp': '<UTC ISO 8601>', 'last_run_id': '<run_id>'}
   - config keys: overlap_buffer_minutes (int, default 15), status_mapping (dict), type_mapping (dict), bridge_env_id (str), run_id (str)
   - Auth failure (CalledProcessError returncode=401): log error, do NOT update checkpoint, re-raise

CONSTRAINTS:
- Stdlib only (no external deps)
- ACLI client injectable via acli_client parameter
- acli_client.get_server_info() is a new method needed on the mock (add to test fixtures)
