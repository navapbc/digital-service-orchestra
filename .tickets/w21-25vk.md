# Shift complexity evaluation from debug-everything to fix-bug post-investigation


## Notes

**2026-03-20T00:35:41Z**


## Context
When a developer runs /dso:debug-everything to fix multiple bugs, the workflow currently evaluates each bug's complexity *before* investigation (Phase 2.5), routing COMPLEX bugs to epics sight-unseen. This was designed before /dso:fix-bug existed as a standalone skill. Now that fix-bug owns the full investigation pipeline (classifying bugs by severity/complexity, then routing through BASIC → INTERMEDIATE → ADVANCED → ESCALATED investigation tiers), complexity evaluation should happen *after* investigation — when the proposed fix's scope is actually known. This prevents premature escalation of bugs that turn out to be simple, and ensures genuinely complex fixes get epic guardrails even if they looked simple at first glance.

This epic supersedes dso-4lfo (Incorporate a specialized Claude debugging workflow into our skill that fixes bugs), which is an underdefined placeholder covering the same domain.

## Success Criteria
1. After fix-bug completes its investigation tier and has a proposed fix, it evaluates the fix's scope using the shared complexity evaluator (skills/shared/prompts/complexity-evaluator.md) — TRIVIAL/MODERATE fixes proceed to the fix phase, COMPLEX fixes create an epic via /dso:brainstorm and stop
2. When fix-bug is dispatched as a sub-agent by debug-everything and determines it needs ADVANCED/ESCALATED investigation (which requires launching its own sub-agents, impossible from within a sub-agent), it returns a structured escalation report — a clearly identifiable result indicating the bug ID, investigation findings so far, the required investigation tier, and the reason orchestrator-level execution is needed — so debug-everything can re-dispatch that bug at orchestrator level
3. Fix-bug includes a belt-and-suspenders fallback: the orchestrator tells fix-bug via prompt that it's running as a sub-agent (primary signal), and fix-bug also checks whether the Agent tool is available before attempting to use it (fallback). Either signal triggers the escalation report path instead of silent failure
4. debug-everything's Phase 2.5 complexity gate is removed — it becomes a triage/dispatch layer that routes all bugs to fix-bug and handles escalation reports by re-dispatching those bugs directly at orchestrator level
5. The existing shared complexity evaluator prompt is reused by fix-bug without duplication — debug-everything no longer references it directly. Note: fix-bug may apply its own routing rules to the evaluator's output (the evaluator classifies; the caller decides what to do with each classification)
6. After delivery, run /dso:debug-everything on the current bug backlog and compare escalation behavior to the previous workflow: track the count of bugs escalated to epics vs. fixed directly — the ratio of false-positive escalations (bugs escalated that turned out TRIVIAL/MODERATE after investigation) should decrease to near zero

## Dependencies
None (dso-4lfo should be closed as superseded)

## Approach
Prompt-based sub-agent context detection (Option A) with belt-and-suspenders fallback on Agent tool availability. fix-bug owns the full investigate → evaluate complexity → fix-or-escalate pipeline. debug-everything becomes a thin triage/dispatch layer.
