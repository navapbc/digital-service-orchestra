# Ticket system v3 — migration from tk and cutover


## Notes

**2026-03-20T03:29:08Z**

## Context
After the new ticket system is feature-complete (dso-0k2k, w21-54wx, w21-bwfw), all existing tickets must be migrated and every reference to tk across skills, scripts, hooks, and documentation must be updated. This is the highest-blast-radius epic in the series — a failed cutover could break every workflow in the plugin. The migration must be reversible: if issues are discovered post-cutover, the old tk system can be restored from git history while fixes are applied.

## Success Criteria
1. A migration script converts all existing .tickets/*.md files into event files on the tickets branch, preserving ticket IDs, status, dependencies, notes, and Jira key mappings
2. All DSO skills, scripts, hooks, and CLAUDE.md references to tk commands are updated to use the new ticket CLI
3. The old .tickets/ directory and tk script are removed from the main branch after successful migration
4. A rollback procedure is documented and tested: if cutover fails, tk can be restored and the tickets branch ignored, with no data loss
5. validate.sh --ci passes after cutover with no regressions
6. After cutover, running /dso:sprint on this project's backlog completes epic selection, task dispatch, and commit workflows using only the new ticket system

## Dependencies
w21-bwfw (Jira bridge — system must be feature-complete before cutover)

## Approach
Migration script reads .tickets/*.md frontmatter and creates corresponding event files. Reference update via grep + sed across all in-scope files. Rollback via git revert of the cutover commit.
