# As a DSO practitioner, a calibrated performance reviewer flags scaling failures and resource exhaustion using bright-line severity rules

## Description

**What**: Create the performance reviewer-delta file and agent definition — a calibrated opus agent that evaluates diffs for performance concerns using bright-line severity rules.
**Why**: Catches performance issues that deterministic tools miss while ensuring only scaling failures and resource exhaustion block commits.
**Scope**:
- IN: reviewer-delta file with 8 AI-advantaged performance criteria (database calls in loops, sequential parallelizable I/O, unbounded accumulation, over-fetching, blocking in async, cache stampede, unnecessary materialization, pool misuse), scrutiny lenses (non-linear complexity, hot paths), bright-line severity rules (it breaks -> critical, it scales -> important, fixed cost -> minor), hard exclusion list, anti-manufacturing directive, build via build-review-agents.sh
- OUT: Dispatch logic, deterministic tool criteria (explicitly excluded)

## Done Definitions

- When this story is complete, a performance reviewer-delta file exists with all 8 performance criteria, bright-line severity rules, and false-positive reduction directives
- When this story is complete, build-review-agents.sh generates the performance reviewer agent from the delta file
- When this story is complete, the performance reviewer outputs findings with severity assigned by the two bright-line tests
- When this story is complete, unit tests written and passing for agent build, severity assignment, and output schema

## Considerations

- [Maintainability] Consistent delta file structure with existing reviewer-delta pattern
