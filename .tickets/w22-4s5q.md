# Create reviewer source fragments (reviewer-base.md and 6 delta files)

Create the source fragment files that build-review-agents.sh will compose into generated agents.

Files to create:
- plugins/dso/docs/workflows/prompts/reviewer-base.md — universal guidance including: output contract (REVIEW_RESULT/REVIEWER_HASH/MIN_SCORE/FINDING_COUNT/FILES), JSON schema, 5-dimension scoring rules, category mapping, no-formatting/linting-exclusion rule, REVIEW-DEFENSE evaluation section, write-reviewer-findings.sh call procedure
- plugins/dso/docs/workflows/prompts/reviewer-delta-light.md — Light tier: single haiku reviewer, reduced checklist (highest-signal checks per dimension)
- plugins/dso/docs/workflows/prompts/reviewer-delta-standard.md — Standard tier: full checklist across all 5 dimensions
- plugins/dso/docs/workflows/prompts/reviewer-delta-deep-correctness.md — Deep Sonnet A: correctness dimension focus
- plugins/dso/docs/workflows/prompts/reviewer-delta-deep-verification.md — Deep Sonnet B: verification dimension focus
- plugins/dso/docs/workflows/prompts/reviewer-delta-deep-hygiene.md — Deep Sonnet C: hygiene + design + maintainability focus
- plugins/dso/docs/workflows/prompts/reviewer-delta-deep-arch.md — Deep Opus: architectural oversight (receives all 3 sonnet findings + full diff)

Content requirements:
- reviewer-base.md must contain the no-formatting/linting-exclusion rule
- Script references in reviewer-base.md use the .claude/scripts/dso shim pattern (no hard-coded plugin paths)
- Delta files contain only tier-specific additions (checklist scope, dimension focus) not present in base
- Delta files are self-contained enough that base+delta produces a coherent agent body

Test-exempt justification: These are pure content files (markdown fragments). They contain no conditional logic, no branching, and no behavioral code. Any test written for their existence would be a change-detector test. Unit tests for the assembled content belong in the build script test task.
