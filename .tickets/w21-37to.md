# Create GitHub Actions outbound bridge workflow

## Description

Create `.github/workflows/outbound-bridge.yml`.

Trigger: push to `tickets` branch with path filter `.tickets/**`.
Concurrency: group `jira-bridge`, `cancel-in-progress: false` (queue, never cancel).
Skip self: `if: github.actor != vars.BRIDGE_BOT_LOGIN` condition on the bridge job.

Steps:
1. actions/checkout@v4 with fetch-depth: 2
2. Set up Python 3.x
3. Install ACLI via cached download (use actions/cache for ACLI jar)
4. Run bridge-outbound.py passing JIRA_URL, JIRA_USER, JIRA_API_TOKEN from GitHub Secrets and BRIDGE_ENV_ID from vars
5. If SYNC events written: git config bridge-bot identity, git add .tickets/, git commit, git push

Security: no token logging — secrets passed as env vars, not CLI args.

TDD Requirement: test-exempt — criterion 3 (GitHub Actions YAML is a static orchestration config file; all behavioral logic is in bridge-outbound.py which is covered by w21-3bqw/w21-dsvz). The 'skip self' conditional (`github.actor != vars.BRIDGE_BOT_LOGIN`) is a GitHub Actions expression evaluated by the GHA engine, not testable in pytest. Integration test (w21-86hr) covers the end-to-end behavior.
