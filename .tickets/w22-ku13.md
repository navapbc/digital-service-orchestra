# As a project maintainer, /dev-onboarding generates a .test-index for associations fuzzy match would miss

## Description

**What**: Add a step to /dso:dev-onboarding that scans the project for source-to-test associations the fuzzy match would miss, generates a .test-index file, and reports coverage. Dogfood against this repo
**Why**: New projects adopting DSO need a one-time deep scan to bootstrap their .test-index with associations that don't follow naming conventions. This also validates the fuzzy match algorithm against real-world naming patterns
**Scope**:
- IN: Scan source and test directories; for each source file, check if fuzzy match finds an associated test; for source files with tests that fuzzy match misses, add an entry to .test-index; report coverage summary (files with fuzzy matches, files with index entries, files with no test coverage); dogfood against this repo to generate its initial .test-index
- OUT: .test-index format/parsing (Story 2); fuzzy match algorithm (Story 1)

## Done Definitions

- When this story is complete, /dso:dev-onboarding includes a step that scans the project and generates a .test-index file with source-to-test mappings that the fuzzy match would miss
  ← Satisfies: "/dev-onboarding includes a step that scans the project for source-to-test associations the fuzzy match would miss"
- When this story is complete, the scanner reports a coverage summary showing files with fuzzy matches, files with index entries, and files with no test coverage
  ← Satisfies: "reports the coverage summary"
- When this story is complete, this repo has a .test-index generated by the scanner, validating the fuzzy match algorithm against real naming conventions
  ← Satisfies: "This is dogfooded against this repo to produce its initial .test-index"
- When this story is complete, unit tests are written and passing for all new or modified logic

## Considerations

- [Reliability] Scanner must distinguish "no test exists" from "test exists but fuzzy match misses it" — only the latter goes in .test-index
