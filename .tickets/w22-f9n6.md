# Add commit-time staleness enforcement for source fragments in pre-commit-review-gate.sh

Implement commit-time staleness enforcement: when source fragment files (reviewer-base.md or reviewer-delta-*.md) are staged, verify that every generated agent file's embedded content hash matches the expected hash of its source inputs. Block the commit if any hash is stale.

Implementation location: plugins/dso/hooks/pre-commit-review-gate.sh (or a helper function sourced from it)

Logic:
1. After detecting staged files, check if any of the following are staged:
   - plugins/dso/docs/workflows/prompts/reviewer-base.md
   - plugins/dso/docs/workflows/prompts/reviewer-delta-*.md
2. If none are staged: skip the staleness check (allow commit)
3. If any are staged: for each of the 6 generated agent files in plugins/dso/agents/code-reviewer-*.md:
   a. Compute expected hash: sha256 of reviewer-base.md + that agent's delta file (same algorithm used by build-review-agents.sh)
   b. Extract embedded hash from the generated file (the 'content-hash:' line)
   c. If hashes do not match: exit non-zero with message referencing build-review-agents.sh as the fix command
4. If all hashes match: allow commit

After implementing, run the tests from task w22-5gj3 to confirm GREEN. Remove the RED marker from .test-index for the staleness tests.
