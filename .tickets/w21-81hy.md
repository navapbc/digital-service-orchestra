# RED: write failing tests for bridge-inbound.py windowed pull, timestamp normalization, CREATE event writing

Write failing tests in tests/scripts/test_bridge_inbound.py for the three core functions of the inbound bridge before implementation.

FILE: tests/scripts/test_bridge_inbound.py (new file)

Test functions to write:

1. test_fetch_jira_changes_builds_correct_jql
   - Given last_pull_ts (UTC ISO 8601 string) and overlap_buffer_minutes (int), fetch_jira_changes() calls ACLI with JQL: updatedDate >= buffered_ts where buffered_ts = last_pull_ts minus overlap_buffer_minutes
   - Mock acli_client.search_issues(jql, start_at, max_results) to return a test fixture list

2. test_fetch_jira_changes_returns_issues_list
   - Returns a list of dicts from ACLI search response; empty results return empty list

3. test_normalize_timestamps_converts_to_utc_epoch
   - normalize_timestamps(issue) converts issue fields created, updated, resolutiondate from Jira ISO 8601 with timezone offset to UTC epoch int
   - Example: 2026-03-21T10:00:00.000+0530 -> UTC epoch for 04:30:00 UTC

4. test_normalize_timestamps_handles_none_fields
   - Fields absent or None in the issue dict are left absent/None (not raised)

5. test_write_create_events_new_ticket_writes_event_file
   - Given a Jira issue dict (new Jira-originated ticket), write_create_events() writes a CREATE event file to .tickets-tracker/<generated_id>/<ts>-<uuid>-CREATE.json
   - Event file must contain: event_type=CREATE, env_id=bridge_env_id, data with Jira fields normalized to UTC timestamps

6. test_write_create_events_skips_existing_local_ticket
   - If a ticket directory already has a SYNC event for this Jira key, write_create_events() skips (idempotency)

Use module loading pattern from test_bridge_outbound.py (importlib, hyphenated filename). Mock acli_client with MagicMock.
