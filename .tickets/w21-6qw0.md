# RED: Write failing tests for SNAPSHOT event handling in ticket-reducer

Write failing pytest tests that assert the ticket-reducer handles SNAPSHOT events correctly and deduplicates events listed in source_event_uuids.

## TDD Requirement
All tests MUST fail (RED) before the SNAPSHOT reducer branch is implemented. Confirm failure with:
  cd $(git rev-parse --show-toplevel) && python3 -m pytest tests/scripts/test_ticket_reducer.py -k 'snapshot' --tb=short -q
Expected: all new snapshot tests fail (AttributeError or AssertionError).

## Implementation Steps
File: tests/scripts/test_ticket_reducer.py (append new test functions)

### test_snapshot_event_restores_compiled_state
- Write a SNAPSHOT event file with event_type='SNAPSHOT' and data={'compiled_state': {ticket_id, ticket_type='task', title='Compacted title', status='closed', author='Alice', created_at=1742605200, comments=[], deps=[], env_id='...', source_event_uuids=['uuid1','uuid2']}}
- Call reduce_ticket(ticket_dir)
- Assert result['title'] == 'Compacted title' and result['status'] == 'closed'

### test_snapshot_plus_post_snapshot_events_applied
- Write SNAPSHOT event restoring state (status='open')
- Write a STATUS event AFTER the snapshot (lexicographically later filename) with uuid NOT in source_event_uuids
- Assert final state has status='closed' (post-snapshot event applied)

### test_snapshot_deduplicates_events_in_source_event_uuids
- Write SNAPSHOT event with source_event_uuids=['dup-uuid-1234']
- Write an additional event file with uuid='dup-uuid-1234' (lexicographically after snapshot)
- Assert the duplicate event's effect does NOT appear in the final state

### test_snapshot_only_ticket_returns_compiled_state
- Write only a SNAPSHOT event (no CREATE, no STATUS events)
- Call reduce_ticket(ticket_dir)
- Assert returns compiled_state dict from snapshot (not None)

### test_cache_invalidation_after_compaction_file_deletion
- Write CREATE event + 3 STATUS events to ticket dir
- Call reduce_ticket() to warm cache
- Delete those 4 files and write a single SNAPSHOT event (simulating compaction)
- Call reduce_ticket() again
- Assert result matches SNAPSHOT compiled_state (cache was invalidated; file count changed)
  Note: this tests the existing cache invalidation logic in w21-f8tg — cache hash covers file sizes,
  so file count change triggers cache miss. This test is RED until SNAPSHOT reducer handles the event type.
