# As a DSO practitioner, oversized diffs are rejected with actionable guidance and merge commits are reviewed efficiently

## Description

**What**: Add diff size thresholds to the classifier/routing pipeline. 300+ lines of non-test non-generated code triggers opus model upgrade at current tier's checklist. 600+ lines triggers rejection with structured message referencing large-diff-splitting-guide.md. Merge commits bypass size limits; review scoped to conflicted + session-modified files only. Create the splitting guide prompt file.

**Why**: Large diffs exhaust reviewer context and degrade review quality. Merge commits shouldn't be penalized for other branches' size. Rejection with actionable guidance incentivizes smaller, more reviewable commits.

**Scope**:
- IN: Diff size thresholds (300/600), model upgrade logic, rejection message, merge commit detection and scoping, large-diff-splitting-guide.md, exceptions for generated code and test-only diffs
- OUT: Classifier scoring factors (w21-jtkr), telemetry (w21-0kt1)

## Done Definitions

- When this story is complete, diffs exceeding 300 lines of non-test non-generated code trigger an opus model upgrade at the current tier's checklist scope
  ← Satisfies: "300+ lines → model upgrade to opus"
- When this story is complete, diffs exceeding 600 lines are rejected with a structured message referencing the splitting guide
  ← Satisfies: "600+ lines → review rejected"
- When this story is complete, diff size rejection applies only to the initial review dispatch, not to re-review passes within the Autonomous Resolution Loop
  ← Satisfies: "Review gate integrity guarantees remain intact" (prevents deadlock)
- When this story is complete, merge commits bypass size limits and review is scoped to conflicted files + session-modified files only
  ← Satisfies: "Merge commits bypass size limits entirely"
- When this story is complete, generated code (migrations, lock files) and test-only diffs bypass size limits at Standard tier
  ← Satisfies: "Exceptions: generated code and test-only diffs bypass size limits"
- When this story is complete, plugins/dso/docs/prompts/large-diff-splitting-guide.md exists with guidance on splitting by concern, by layer, and keeping tests with code
  ← Satisfies: "Structured message referencing large-diff-splitting-guide.md"
- Unit tests written and passing for all new or modified logic

## Considerations

- [Reliability] Merge commit scoping depends on git merge-base — must handle edge cases (shallow clones, octopus merges)
- [Reliability] 600-line rejection during the resolution loop would create a deadlock — size limits must apply to initial review only

**Escalation policy**: Proceed unless a significant assumption is required to continue — one that could send the implementation in the wrong direction. Escalate only when genuinely blocked without a reasonable inference. Document all assumptions made without escalating.
