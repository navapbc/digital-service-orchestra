# Integrate search into planning

Websearch and webfetch should be used during planning when it would significantly improve the quality of our solutions or confidence in our solutions. We should work to develop a bright line for when websearch should be used. Examples include:

- Searching for examples of plug-in project structure to see what conventions work. This pattern applies to many cases where documentation on specific functionality is lacking and working examples can increase confidence.
- Searching for best practices and expert advice when a task involves ambiguous criteria or requires nuanced judgement. For example, when code should require unit tests vs not.
- Searching for a better option when faced with difficult tradeoffs. When no option perfectly meets requirements, there may be an option we aren't considering.
