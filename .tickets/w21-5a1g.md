## Test Coverage Required (minimum 8 test cases)

1. Bare configured `commands.test_unit` command is blocked (hook exit 2, emits Structured Action-Required Block)
2. Configured `commands.test_unit` command with `cd app &&` prefix is blocked (Matching Contract: strip cd-prefix)
3. Configured `commands.test_unit` command with additional arguments passes through (not blocked — substring must not match)
4. Unconfigured command (e.g., `pytest tests/unit/test_foo.py`) passes through
5. `validate.sh` invocation passes through (not intercepted — safety allowlist)
6. `test-batched.sh` invocation passes through (not intercepted — safety allowlist)
7. `blocked_test_command` JSONL telemetry entry is written to `$ARTIFACTS_DIR/hook-telemetry.jsonl` on interception
8. Non-Bash tool call (e.g., Read tool with tool_name != "Bash") passes through immediately (fast early-exit)
9. Empty/missing `commands.test_unit` config key causes pass-through (graceful degradation)
10. Commands with shell operators (`&&`) where the blocked value appears as a complete segment after operator splitting are blocked
11. CLAUDE_PLUGIN_ROOT unset: hook falls back to BASH_SOURCE-relative path for validate.sh resolution; RUN: line in output contains an absolute path (not a relative one)

## Implementation Notes

- Follow the existing hook function pattern in `pre-bash-functions.sh`:
  - ERR trap for fail-open (`return 0` on unexpected errors), log to `$HOME/.claude/hook-error-log.jsonl`
  - `parse_json_field` for JSON parsing (no jq — see `deps.sh`)
  - Fast early-exit guard: check `INPUT` for `test` substring before full JSON parse (performance — hook runs on every Bash call)
  - Use `$CLAUDE_PLUGIN_ROOT/scripts/read-config.sh` (or derive plugin root from `BASH_SOURCE`) to read `commands.test_unit` and `commands.test_e2e`
  - `WORKFLOW_CONFIG_FILE` env var must be honored for test isolation (delegated to `read-config.sh`)
- Matching Contract implementation (from epic w21-p7aa):
  1. Split input command on shell operators (`&&`, `||`, `;`, `|`)
  2. From each resulting segment, strip all leading `cd <path> &&` segments (zero or more)
  3. Trim leading/trailing whitespace from each resulting segment
  4. Check if any resulting segment equals a configured `commands.test_unit` or `commands.test_e2e` value verbatim (complete token sequence, not substring)
  5. Commands with additional arguments appended after the configured value do NOT match
- Structured Action-Required Block (emit to stdout, hook exit 2):
  ```
  ACTION REQUIRED: tests incomplete
  RUN: <CLAUDE_PLUGIN_ROOT>/scripts/validate.sh --ci
  DO NOT proceed without completing all test batches.
  ```
  Resolve `CLAUDE_PLUGIN_ROOT` at runtime from `$CLAUDE_PLUGIN_ROOT` env var or via `BASH_SOURCE`-relative path.
- Telemetry: append JSONL entry to `$ARTIFACTS_DIR/hook-telemetry.jsonl` (create dir if needed):
  `{"ts": "<ISO8601>", "event": "blocked_test_command", "command": "<intercepted command>"}`
  Use `python3` for JSON serialization (existing pattern — see `post-functions.sh`).
- The `validate.sh` and `test-batched.sh` allowlist must be checked BEFORE the matching contract check.
- Test isolation pattern: use `WORKFLOW_CONFIG_FILE` env var pointing at a temp config file with a controlled `commands.test_unit` value (e.g., `make test-unit-only`). Also set `ARTIFACTS_DIR` to a temp dir.
