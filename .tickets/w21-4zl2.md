# Implement inbound STATUS event writer and jira_sync_status relationship rejection in bridge-inbound.py

## Description

Implement two features in `plugins/dso/scripts/bridge-inbound.py`:
1. `write_status_event()` — write bridge-authored STATUS event file when Jira status changes, enabling bidirectional flap detection
2. `jira_sync_status: rejected` — persist local rejection state when Jira rejects a relationship; never remove local relationship

**TDD requirement:** Tests in w21-bef9 must be RED before starting. Run `python3 -m pytest tests/scripts/test_bridge_inbound.py -k 'write_status_event or rejection' --tb=line -q` and confirm failures. Then implement until all tests pass GREEN.

**Implementation steps:**

1. Add `write_status_event(ticket_id: str, status: str, ticket_dir: Path, bridge_env_id: str, run_id: str = "") -> Path` to bridge-inbound.py:
   - Creates `{ts}-{uuid}-STATUS.json` in ticket_dir with:
     - event_type: "STATUS"
     - env_id: bridge_env_id (marks as bridge-authored)
     - data: {"status": status}
     - timestamp, uuid, run_id
   - Returns path to written file

2. Integrate `write_status_event()` into `process_inbound()`:
   - After fetching and normalizing Jira issues, for each issue with a mapped status:
     - Read the current local compiled status (via ticket-reducer.py) for that ticket
     - If Jira status differs from local compiled status: call `write_status_event()`
     - This enables the outbound flap detector to see bidirectional STATUS history

3. Add `persist_relationship_rejection(ticket_id: str, relationship: str, reason: str, tickets_root: Path) -> Path`:
   - Write a `.jira-sync-status` file to the ticket dir: `{"jira_sync_status": "rejected", "relationship": relationship, "reason": reason}`
   - The local relationship (in the ticket's links/deps) is NEVER modified — only the status file is written

4. Integrate relationship rejection into `process_inbound()`:
   - When `acli_client.set_relationship()` raises an error (Jira schema disallows the relationship), call `persist_relationship_rejection()` instead of removing the local relationship
   - Write a BRIDGE_ALERT event with reason "Jira rejected relationship: {reason}"

**Files to modify:** plugins/dso/scripts/bridge-inbound.py
