# Add diff-size threshold logic to review-complexity-classifier.sh

Implement diff-size threshold detection in plugins/dso/scripts/review-complexity-classifier.sh. This is the core implementation task.

TDD Requirement: RED tests in tests/hooks/test-review-complexity-classifier.sh (created in T1/w22-r2eh) must fail before this task and pass after.

Implementation:

1. Add _count_non_test_non_generated_lines() function:
   - Count added lines (lines starting with '+' excluding '+++') in non-test, non-generated files
   - Test file detection: use existing is_test_file() helper
   - Generated file detection: add is_generated_file() function matching migrations/, *.lock, package-lock.json, yarn.lock, poetry.lock, go.sum, *.pb.go patterns
   - Exclude .tickets/* files (already handled by diff_lines)
   - Count only added/modified lines ('+' prefix), not removed lines

2. Add merge-commit detection:
   - Check for MERGE_HEAD env var (set by test fixtures) OR presence of $(git rev-parse --git-dir)/MERGE_HEAD file
   - If merge commit detected: set IS_MERGE_COMMIT=true

3. Add diff-size classification:
   - SIZE_LINE_COUNT=$(_count_non_test_non_generated_lines)
   - If IS_MERGE_COMMIT=true: MODEL_OVERRIDE=null, SIZE_REJECTION=false
   - Elif SIZE_LINE_COUNT >= 600 AND diff is NOT test-only AND NOT generated-only: SIZE_REJECTION=true, rejection_reason referencing plugins/dso/docs/prompts/large-diff-splitting-guide.md
   - Elif SIZE_LINE_COUNT >= 300: MODEL_OVERRIDE=opus, SIZE_REJECTION=false
   - Else: MODEL_OVERRIDE=null, SIZE_REJECTION=false

4. Add is_test_only_diff() helper:
   - Returns true if ALL source lines in diff belong to test files (is_test_file())
   - Used to bypass size limits for test-only diffs

5. Update JSON output (stdout) to include new fields:
   - Add size_line_count, model_override (string|null), size_rejection (bool), rejection_reason (string|null)
   - Maintain backward compat: all existing fields unchanged

6. Update telemetry JSONL to include new fields (same fields as stdout)
