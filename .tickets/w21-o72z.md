# As a developer, I can list, transition, and comment on tickets with full CRUD support

## Description

**What**: Complete the CRUD command set: `ticket list` (reducer compiles all tickets), `ticket transition` (STATUS event), `ticket comment` (COMMENT event). Add ghost ticket prevention.

**Why**: Completes the basic command surface needed for any ticket workflow.

**Scope**:
- IN: ticket list, ticket transition, ticket comment, ghost ticket prevention, edge case handling for empty/corrupt ticket directories
- OUT: caching (w21-f8tg), compaction (w21-q0nn), fsck (w21-njch)

## Done Definitions

- When this story is complete, `ticket list` compiles all tickets via the reducer and returns their current state
  ← Satisfies: "ticket list works correctly"
- When this story is complete, `ticket transition` appends a STATUS event and auto-commits
  ← Satisfies: "ticket transition works correctly"
- When this story is complete, `ticket comment` appends a COMMENT event and auto-commits
  ← Satisfies: "ticket comment works correctly"
- When this story is complete, non-CREATE commands verify the target ticket has a CREATE event and fail fast with a clear error if not
  ← Satisfies: "Non-CREATE write commands verify the target ticket directory exists and contains a CREATE event"
- When this story is complete, the reducer handles ticket directories with zero valid events by returning an error state, not crashing
  ← Satisfies: adversarial review — prevents cache from caching crash state
- When this story is complete, ghost prevention defines behavior for corrupt CREATE: ticket is flagged as needing fsck repair rather than silently blocking all operations
  ← Satisfies: adversarial review — resolves S2/S5 conflict on corrupt CREATE
- When this story is complete, unit tests are written and passing for all new logic

## Considerations

- [Maintainability] Reuse the shared write-commit function from w21-ablv
- [Reliability] Ghost prevention + corrupt event handling (w21-njch) have conflicting defaults — define the interaction explicitly
