# Implementation planning tradeoffs

The implementation plan skill should consider whether there are multiple approaches to implementing the selected functionality. If confidence is high that one approach is superior, implementation plan should proceed with that approach. If multiple approaches present distinct advantages and the agent lacks high confidence that one approach is superior, the decision on which approach to use should be escalated to the user. When an escalation occurs, the user should be provided with detailed descriptions of each approach that include pros, cons, and tradeoffs.
