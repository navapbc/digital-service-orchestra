# As a DSO practitioner, every classification decision is logged and post-deployment calibration signals are defined

## Description

**What**: Add JSONL telemetry logging to the classifier — every invocation appends a JSON object to $ARTIFACTS_DIR/classifier-telemetry.jsonl. Define validation signal thresholds, expected tier distribution baseline, and breach response protocol.

**Why**: The classifier is a single point of failure for quality routing. Telemetry enables post-deployment calibration and early detection of miscalibration.

**Scope**:
- IN: JSONL telemetry output from classifier, validation signal thresholds (tier distribution, Light-tier finding rate, CI failure rate by tier), breach response protocol, expected healthy baseline documentation
- OUT: Enriching the telemetry with re-review iteration counts (Epic B)

## Done Definitions

- When this story is complete, every classifier invocation appends a valid JSON object to classifier-telemetry.jsonl containing all factor scores, computed_total, selected_tier, and staged file paths
  ← Satisfies: "Every classification decision appends one JSON object to classifier-telemetry.jsonl"
- When this story is complete, validation signal thresholds are documented: tier distribution baseline (~50-60% Light, 30-40% Standard, 5-15% Deep), Light-tier finding rate (<10% critical/important), post-merge CI failure rate by tier
  ← Satisfies: "After 30 commits: compute tier distribution"
- When this story is complete, breach response protocol is documented: P1 bug ticket, adjust classifier, re-validate against sample
  ← Satisfies: "Breach response: create P1 bug ticket, adjust classifier, re-validate"
- Unit tests written and passing for all new or modified logic

## Considerations

- [Maintainability] Define the relationship between classifier-telemetry.jsonl and the existing review-gate-telemetry.jsonl — correlation key or explicit independence

## Escalation Policy

**Escalation policy**: Proceed unless a significant assumption is required to continue — one that could send the implementation in the wrong direction. Escalate only when genuinely blocked without a reasonable inference. Document all assumptions made without escalating.
