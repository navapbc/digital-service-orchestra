# Implement tombstone resolution in tk dep tree

Implement tombstone-based dep resolution in the tk dep tree command. When a dep ID is not found in statuses[] (the 'missing' path in the awk dep tree traversal, currently line 716-722 of tk), check for a tombstone file at .tickets/archive/tombstones/<id>.json. If tombstone exists: load its id, type, final_status; display the dep as '[archived: <final_status> (<type>)]' instead of '[missing - treated as satisfied]'. Tombstone deps are still treated as satisfied for ready_to_work computation (same as current missing behavior). Implementation: (1) in the awk BEGIN block, read tombstone directory and load tombstone data into tombstone_status[] and tombstone_type[] arrays, (2) in the print loop for missing deps (line 801-806 of tk), check if id is in tombstone arrays and use '[archived: <status> (<type>)]' label. Fallback: if no tombstone file, keep existing '[missing - treated as satisfied]' label. TDD Requirement: Run bash tests/scripts/test-dep-tree-tombstone-resolution.sh — all 4 tests must pass GREEN.
