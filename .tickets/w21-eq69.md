# IMPL: Create record-test-exemption.sh

Implement plugins/dso/hooks/record-test-exemption.sh — a protected script that proves a test is inherently slow and records an exemption.

BEHAVIOR:
- Accept exactly one argument: <test-node-id> (pytest node id or shell test path)
- Run the test in isolation using a 60-second timeout (via get_timeout_cmd from deps.sh, or 'timeout 60')
- If the test EXCEEDS the timeout (timeout exit code, typically 124): write exemption entry and exit 0
- If the test completes within 60s: do NOT write exemption, print error 'ERROR: test completed within 60s — not eligible for exemption', exit 1
- If no argument provided: print usage, exit 1

EXEMPTION FILE FORMAT (pipe-delimited, single-parsing-path):
  Path: $(get_artifacts_dir)/test-exemptions
  One entry per line: node_id=<id>|threshold=60|timestamp=<ISO8601-UTC>
  Idempotency: overwrite existing entry for same node_id (filter + append)

ENVIRONMENT:
  RECORD_TEST_EXEMPTION_RUNNER — override test runner (for testing)
  WORKFLOW_PLUGIN_ARTIFACTS_DIR — override artifacts dir (for testing)
  CLAUDE_PLUGIN_ROOT — DSO plugin root
