# As a DSO adopter, CI workflows are auto-detected and ci.* config is consolidated and confirmed

## Description

**What**: Using detection script results, auto-detect CI workflow names and job display names from .github/workflows/. Consolidate merge.ci_workflow_name into ci.workflow_name. Present detected values to the user for confirmation. Analyze existing workflows for lint/format/test guard steps before offering to add them.
**Why**: CI config should be derived from what already exists, not manually entered. The merge.ci_workflow_name / ci.* split is confusing — one ci.* namespace is cleaner.
**Scope**:
- IN: ci.workflow_name (consolidated from merge.ci_workflow_name), ci.fast_gate_job, ci.fast_fail_job, ci.test_ceil_job, ci.integration_workflow — all auto-detected and confirmed. Update merge-to-main.sh to read ci.workflow_name instead of merge.ci_workflow_name (with backward compat fallback). Update CONFIGURATION-REFERENCE.md.
- OUT: Template file copying/merging (separate story), command configuration (separate story)

## Done Definitions

- When this story is complete, CI job names and workflow name are auto-detected from .github/workflows/ YAML files and presented to the user for confirmation
  ← Satisfies: "CI job names and workflow name are auto-detected from .github/workflows/ and confirmed by the user"
- When this story is complete, merge.ci_workflow_name is consolidated into ci.workflow_name with a backward-compatible fallback in merge-to-main.sh
  ← Satisfies: "ci.* keys (consolidated from merge.ci_workflow_name) are prompted when project context indicates they are relevant"
- When this story is complete, validate-config.sh known-keys list includes ci.workflow_name (and retains merge.ci_workflow_name for backward compat)
  ← Satisfies: cross-story integration (adversarial review finding)
- When this story is complete, unit tests written and passing for all new or modified logic

## Considerations

- [Maintainability] merge-to-main.sh must fall back to merge.ci_workflow_name if ci.workflow_name is absent, for backward compatibility with existing configs
- [Reliability] CI YAML parsing should handle multi-file workflows and non-standard job naming
- [Maintainability] When fallback to merge.ci_workflow_name is triggered, emit a deprecation warning to stderr suggesting migration to ci.workflow_name
