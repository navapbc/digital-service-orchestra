# As a host project developer, I can explicitly map source files to unconventionally-named tests via .test-index

## Description

**What**: Add .test-index file support to the test gate — a line-based mapping file at the repo root that explicitly associates source files with test files that don't follow naming conventions
**Why**: Fuzzy matching covers standard conventions but misses E2E tests, integration tests, and cross-cutting test files named differently from their source files. The .test-index provides explicit coverage for those cases
**Scope**:
- IN: Parse .test-index (format: source/path.ext: test/path1.ext, test/path2.ext); merge index associations with fuzzy matches at gate time; auto-prune stale entries (nonexistent test files) during pre-commit and stage the modified index; handle edge cases (missing index = no-op, empty right side = remove line)
- OUT: Generating the .test-index (Story 3 handles that via /dev-onboarding); fuzzy match logic (Story 1)

## Done Definitions

- When this story is complete, a .test-index file at the repo root can map source files to test files, and staging a mapped source file adds those tests to the required set
  ← Satisfies: "A .test-index file can explicitly associate source files with unconventionally-named tests"
- When this story is complete, during pre-commit, any .test-index entry whose test file does not exist on disk is removed from the index and the modified index is staged automatically
  ← Satisfies: "Stale .test-index entries are automatically pruned during commit"
- When this story is complete, if the .test-index file does not exist, the gate proceeds without error (fail-open for missing index)
  ← Satisfies: "The test gate requires zero configuration for projects that follow standard test naming conventions"
- When this story is complete, unit tests are written and passing for all new or modified logic

## Considerations

- [Reliability] Self-healing pruning modifies .test-index on disk during pre-commit and auto-stages — must handle concurrent commits and partial writes gracefully
- [Maintainability] Two test association mechanisms (fuzzy match + index) merge at gate level — union logic should be clean
- [Reliability] Auto-staging a modified .test-index during pre-commit invalidates the diff hash captured by the review gate — .test-index must be added to compute-diff-hash.sh's exclusion list or review-gate-allowlist.conf to prevent a hash mismatch loop that blocks every commit with stale entries
