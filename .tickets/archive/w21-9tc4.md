# Bug: sub-agent wrote discovery file under .claude/ (protected folder) instead of artifacts dir

A sub-agent executing task w21-57k1 wrote a discovery file to .claude/artifacts/agent-discoveries/w21-57k1.json. The .claude/ directory is protected in this repo (review-gate blocks writes). Discovery files should be written to the artifacts dir under /tmp/ (via get_artifacts_dir()), or reported as a ticket via tk create, or written to the blackboard. The task-execution.md prompt template references CLAUDE_PLUGIN_ROOT for the discovery path, but the sub-agent resolved it to .claude/ which is the host project's protected directory. Fix: update the discovery file template in task-execution.md to use get_artifacts_dir() directly without relying on CLAUDE_PLUGIN_ROOT.
