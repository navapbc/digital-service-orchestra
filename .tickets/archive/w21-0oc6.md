# Bug: review gate blocks merge commits from main into worktree — pre-commit-review-gate treats incoming main changes as requiring review


## Notes

**2026-03-21T03:53:56Z**

When merge-to-main.sh syncs main into the worktree (git merge origin/main), the resulting merge commit includes files changed on main since the worktree branched off. The pre-commit review gate sees these files as non-allowlisted and requires a review — but they were already reviewed and merged on main. The review gate's MERGE_HEAD detection (documented in CLAUDE.md) may not be handling this case correctly, or the version bump conflict resolution creates a state where the gate can't distinguish worktree changes from incoming main changes.

**2026-03-21T03:55:46Z**

Investigation log from session worktree-20260320-194801:

1. merge-to-main.sh sync phase: git merge origin/main into worktree → CONFLICT in plugin.json (version 0.12.10 vs 0.13.0)
2. Resolved conflict (took main's 0.13.0, bumped to 0.13.1), git add, git commit --no-edit
3. BLOCKED by pre-commit review gate: 'Diff hash mismatch — code changed since review was recorded'. The merge commit included 13 files from main (CLAUDE.md, validate.sh, test-batched.sh, etc.) that were not in our review.
4. git merge --abort, tried merge-to-main.sh --phase=merge (succeeded) then --phase=push (failed — tried pushing worktree branch instead of main)
5. git rebase origin/main → same version conflict in plugin.json → resolved (theirs + bump to 0.13.1) → rebase --continue succeeded
6. merge-to-main.sh full run succeeded after rebase (sync found no more conflicts)

Root cause: The review gate treats merge commit's incoming files from main as requiring review. This only manifests when main has diverged with non-allowlisted file changes since the worktree branched off. The rebase workaround avoids the merge commit entirely.

Workaround: rebase onto origin/main before running merge-to-main.sh when version conflicts exist.

**2026-03-21T16:07:40Z**

Fixed: pre-commit-review-gate.sh now filters incoming-only files during merge commits (commit 676ca3b)
