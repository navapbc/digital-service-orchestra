# Adjust Value reviewer validation_signal to remove external feedback requirement


## Notes

**2026-03-20T01:26:12Z**

## Context
The brainstorm fidelity reviewer for Value (Senior Product Manager perspective) currently lists external user feedback mechanisms — usability testing, support ticket volume decrease — as valid validation signals. For a developer tooling plugin with no external user base, these are unachievable and score epics down for lacking something outside the development workflow's scope. The validation_signal dimension should be scoped to signals the development team can actually observe: dogfooding, operational metrics, before/after workflow comparisons, and staged rollouts.

## Success Criteria
1. The validation_signal dimension in skills/brainstorm/docs/reviewers/value.md no longer lists external feedback mechanisms (usability testing, support ticket volume decrease, A/B tests) as valid signals
2. The dimension lists internal/dogfooding signals as the valid spectrum: before/after workflow comparisons, operational metrics (error rates, cycle times, latency), dogfooding observations, staged rollout with rollback criteria
3. The core principle is preserved: 'shipping is not the same as solving the problem' — the dimension still requires a concrete validation mechanism, just scoped to what the team can observe
4. After delivery, re-running the brainstorm fidelity review on an existing epic spec (e.g., w21-25vk) produces a validation_signal score of 4+ without requiring external feedback mechanisms

## Dependencies
None

## Approach
Direct edit of the validation_signal dimension description and examples in skills/brainstorm/docs/reviewers/value.md.
