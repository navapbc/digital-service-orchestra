# Implement ticket-lib.sh shared write_commit_event helper (atomic write + flock)


## Description

Implement `plugins/dso/scripts/ticket-lib.sh` — a shared library sourced by `ticket-create.sh` and future write commands (w21-o72z). This prevents drift between commands as described in Story Considerations.

### Implementation steps:
1. Create `plugins/dso/scripts/ticket-lib.sh` with function `write_commit_event`:

```bash
# write_commit_event <ticket_id> <temp_event_json_path>
# Atomically writes the event file and commits it to the tickets branch.
# Args:
#   ticket_id: the ticket directory name (e.g., w21-ablv)
#   temp_event_json_path: path to the fully-constructed JSON event file (temp file)
# The function:
#   1. Determines final filename: <timestamp>-<uuid>-<TYPE>.json (reads type from JSON)
#   2. Creates ticket dir: mkdir -p .tickets-tracker/<ticket_id>
#   3. Atomic write: mv <temp_file> .tickets-tracker/<ticket_id>/<final_filename>
#      (temp file and target must be on same filesystem — use mktemp in .tickets-tracker/)
#   4. Acquires flock on .tickets-tracker/.ticket-write.lock (timeout 30s, max 2 retries)
#   5. Runs: git -C .tickets-tracker add <ticket_id>/<final_filename>
#   6. Runs: git -C .tickets-tracker commit -m "ticket: <event_type> <ticket_id>"
#   7. Releases flock
#   8. On flock timeout after max retries: exit 1 with error "flock: could not acquire lock after <N>s"
```

2. Lock file location: `.tickets-tracker/.ticket-write.lock` (global per-worktree, not per-ticket)
   - This satisfies the flock contract requirement (w21-mtvm defines this)

3. Flock timeout budget (per Done Definition 14 in epic):
   - Max flock wait: 30 seconds per attempt
   - Max retries: 2 (total max wait: 60 seconds)
   - Fail fast with clear error if budget exhausted

4. `gc.auto = 0` enforcement: verify on each call, set if not already set (idempotent guard)

### Constraints:
- Temp file must be created in `.tickets-tracker/` (same filesystem as target for atomic rename)
- `git add <specific-file>` only — never `git add -A` or `git add .`
- Library is sourced (`. ticket-lib.sh`), not executed directly
- All JSON reading via python3 (to extract event_type from temp file for naming)

## Gap Analysis Amendments (from Step 6)

**AC Amendment — same-filesystem temp file:**
- [ ] Temp file for atomic write is created in `.tickets-tracker/` directory (not in /tmp or other filesystem)
  Verify: `grep -q '\.tickets-tracker.*mktemp\|mktemp.*\.tickets-tracker' $(git rev-parse --show-toplevel)/plugins/dso/scripts/ticket-lib.sh`
  Rationale: `mv` is only atomic when source and destination are on the same filesystem. If the temp file is in /tmp and .tickets-tracker/ is on a different filesystem, mv becomes a copy+delete (not atomic), defeating the crash-safety guarantee.
