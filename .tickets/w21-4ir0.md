# IMPL-CI-WORKFLOWS: Implement CI workflow YAML analysis with confidence degradation

## Implementation — CI workflow analysis detection category

Add CI workflow analysis to plugins/dso/scripts/project-detect.sh.

## Detection category: CI workflow analysis

Scan PROJECT_DIR/.github/workflows/*.yml and *.yaml files. If directory absent, emit empty/false values and exit gracefully.

### Keys to emit

ci_workflow_names=<comma-separated workflow file basenames without extension (e.g., ci,release,lint)>
ci_workflow_test_guarded=true|false  (true if any step run contains pytest/go test/npm test/cargo test/make test)
ci_workflow_lint_guarded=true|false  (true if any step run contains ruff/eslint/golangci-lint/cargo clippy/make lint)
ci_workflow_format_guarded=true|false  (true if any step run contains ruff format/prettier/gofmt/make format)
ci_workflow_confidence=high|low  (high: workflows found and analyzed; low: no .github/workflows dir or empty)

### Implementation approach

Use grep-based heuristics (avoids PyYAML dependency, consistent with jq-free hook pattern):

1. Enumerate .yml/.yaml files in .github/workflows/, build comma-separated names list
2. Cat all workflow files and grep with -iE for test/lint/format patterns
3. Set ci_workflow_confidence=high if workflow files were found and readable

```bash
ci_workflow_names=""
ci_workflow_test_guarded="false"
ci_workflow_lint_guarded="false"
ci_workflow_format_guarded="false"
ci_workflow_confidence="low"

if [[ -d "$PROJECT_DIR/.github/workflows" ]]; then
  for f in "$PROJECT_DIR/.github/workflows"/*.yml "$PROJECT_DIR/.github/workflows"/*.yaml; do
    [[ -f "$f" ]] || continue
    bname="${f##*/}"; bname="${bname%.yml}"; bname="${bname%.yaml}"
    ci_workflow_names+="${bname},"
  done
  ci_workflow_names="${ci_workflow_names%,}"
  wf_content=$(cat "$PROJECT_DIR/.github/workflows"/*.yml "$PROJECT_DIR/.github/workflows"/*.yaml 2>/dev/null || true)
  if [[ -n "$wf_content" ]]; then
    ci_workflow_confidence="high"
    echo "$wf_content" | grep -qiE 'pytest|go test|npm test|cargo test|make test' && ci_workflow_test_guarded="true"
    echo "$wf_content" | grep -qiE 'ruff|eslint|golangci-lint|cargo clippy|make lint' && ci_workflow_lint_guarded="true"
    echo "$wf_content" | grep -qiE 'ruff format|prettier|gofmt|make format' && ci_workflow_format_guarded="true"
  fi
fi
echo "ci_workflow_names=$ci_workflow_names"
echo "ci_workflow_test_guarded=$ci_workflow_test_guarded"
echo "ci_workflow_lint_guarded=$ci_workflow_lint_guarded"
echo "ci_workflow_format_guarded=$ci_workflow_format_guarded"
echo "ci_workflow_confidence=$ci_workflow_confidence"
```

Note: The story mentions 'YAML workflow analysis heuristics' as the confidence-Medium reason. This grep approach is intentional and avoids PyYAML dependency.
