# As a developer using /implementation-plan, I get guidance to name tests for fuzzy-match compatibility

## Description

**What**: Update /dso:implementation-plan skill to recommend test filenames that follow fuzzy-matchable conventions and note that unconventional names require a .test-index entry
**Why**: Upstream enforcement prevents the false-negative gap from growing — by the time code is committed, the test association already exists either via naming convention or .test-index
**Scope**:
- IN: Add guidance to the implementation-plan skill's task creation logic; when proposing test file names, validate they would be caught by fuzzy match; if not, include a .test-index entry as an acceptance criterion
