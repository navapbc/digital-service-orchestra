# As a practitioner invoking /dso:sprint, I receive complexity classifications from a dedicated complexity-evaluator agent

## Description

**What**: Create the `dso:complexity-evaluator` dedicated agent definition and update sprint's two complexity evaluation dispatches (epic evaluator and story evaluator) to use the named agent instead of loading prompt content into a general-purpose task.
**Why**: Promoting classification logic to a dedicated agent definition — where it becomes the agent's system prompt and core identity — improves output format compliance and reduces malformed JSON re-dispatch cycles. Sprint is the walking skeleton caller because it exercises both the epic evaluator (SIMPLE/MODERATE/COMPLEX) and story evaluator (TRIVIAL/MODERATE/COMPLEX) paths.
**Scope**:
- IN: Create `plugins/dso/agents/complexity-evaluator.md` with YAML frontmatter and markdown body. Update sprint epic evaluator dispatch. Update sprint story evaluator dispatch. Both switch to `subagent_type: dso:complexity-evaluator`.
- OUT: Other callers (brainstorm, fix-bug, debug-everything — w21-cw8j). Resolve-conflicts (w21-kezk). Shared rubric file lifecycle (w21-nica).
