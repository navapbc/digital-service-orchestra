# Bug: preplanning uses ambiguous confirmation questions where yes/no answers conflict


## Notes

**2026-03-21T17:03:43Z**

## Problem

Preplanning's final review prompt (Phase 4 Step 5) asks two questions where 'yes' means approval for the first but disapproval for the second:

> Does this plan fully capture your vision? Should we adjust any priorities before I finalize?

Answering 'yes' is ambiguous — it could mean 'yes it captures my vision' (approve) or 'yes adjust priorities' (request changes). The user cannot give a clear single-word answer.

## Root Cause

The SKILL.md Step 5 template contains contradictory confirmation questions. The AskUserQuestion tool is available and provides unambiguous option selection, but the prose questions preceding it create confusion.

## Expected Behavior

Use a single unambiguous confirmation — either one clear question ('Do you approve this plan?') or rely solely on AskUserQuestion with labeled options. Never pair two questions where the same answer (yes/no) has opposite meanings.
