# As a DSO practitioner, a security blue team agent triages red team findings with ticket context to dismiss, downgrade, or sustain

## Description

**What**: Create the security blue team reviewer-delta file and agent definition — a context-aware opus agent that triages red team findings with full ticket context, with authority to dismiss, downgrade, or sustain findings.
**Why**: Reduces false positives by evaluating red team findings against the actual intent of the change, preventing invalid findings from blocking valid code.
**Scope**:
- IN: reviewer-delta file with triage logic (dismiss/downgrade/sustain), ticket context integration, output schema showing disposition and rationale for each red team finding, build via build-review-agents.sh
- OUT: Red team criteria, dispatch logic, deterministic tool criteria

## Done Definitions

- When this story is complete, a security blue team reviewer-delta file exists with dismiss/downgrade/sustain triage logic
- When this story is complete, build-review-agents.sh generates the security blue team agent from the delta file
- When this story is complete, the blue team agent receives red team findings (per the schema contract from Story 4) and full ticket context, and outputs a disposition (dismiss/downgrade/sustain) with rationale for each finding
- When this story is complete, only findings sustained or downgraded at critical/important severity block the commit; dismissed and minor findings create tracking tickets
- When this story is complete, unit tests written and passing for agent build, triage logic, and output schema

## Considerations

- [Reliability] Schema contract between red team output and blue team input must be explicit — consumed from Story 4's contract artifact
- [Performance] Blue team runs serially after red team — review latency impact on commit workflow
