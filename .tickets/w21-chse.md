# Add hypothetical scenario consideration to epic planning

When creating an epic using brainstorm, the skill should conduct a red team scenario analysis alongside the current fidelity review. This analysis should think through possible usage scenarios and identify gaps that the proposed epic hasn't considered. This should remain at a high level suitable to an epic, not a detailed implementation level. Examples include considering timeouts, race conditions, conflicts, and a user trying to perform operations out of order.

Our goal is to harden the epic in a way that creates a more reliable solution without forcing the epic to go into implementation details. We are trying to understand the different ways our desired functionality could be used, and make sure we have clearly accounted for them.
