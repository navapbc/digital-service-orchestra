# As a DSO practitioner, overlay reviewers are dispatched in parallel or serially based on defense-in-depth classification signals

## Description

**What**: Implement the defense-in-depth dispatch: parallel overlay when deterministic signal fires, serial overlay when tier reviewer raises flag, no overlay when neither fires.
**Why**: Connects classifier signals and reviewer flags to overlay agent invocation with graceful degradation.
**Scope**:
- IN: Dispatch logic in REVIEW-WORKFLOW.md, parallel/serial branching, graceful degradation on overlay failure (timeout/malformed output does not block commit)
- OUT: Classifier extension, reviewer agent definitions, checklist items

## Done Definitions

- When this story is complete, a deterministic classifier signal triggers the corresponding overlay in parallel with the tier reviewer
- When this story is complete, a tier reviewer classification flag (without deterministic signal) triggers the overlay serially after tier review completes
- When this story is complete, when neither signal fires, no overlay runs
- When this story is complete, overlay agent failure (timeout, malformed output) does not block the commit — the tier review proceeds normally
- When this story is complete, unit tests written and passing for all dispatch logic branches

## Considerations

- [Reliability] Overlay dispatch failure needs graceful degradation — overlay failure must not block the commit
- [Architecture] Overlay dispatch must be handled at the orchestrator level (not within the review sub-agent) to avoid violating the two-level nesting prohibition in CLAUDE.md rule 23. The serial path requires tier reviewer classification flags to be returned to the orchestrator before overlay dispatch.
